Synthetic Healthcare Data Generator

Upload a data dictionary and generate realistic mock records for healthcare administration, clinical research, medical operations, and public health surveillance. The inference engine uses field names, comments, and data types to assign healthcare-specific mock-data rules.

⚠️
Synthetic data only — do not upload real patient data or PHI.

This tool is intended for mock data generation, development, demos, analytics testing, and training. Files are processed in the browser, but this app is not a legal HIPAA compliance product and should not be used with protected health information.

Configuration

Accepted formats: CSV, JSON, XLS, XLSX. Expected columns include name, comment/description, and type.
The profile changes defaults for diagnoses, departments, workflow fields, and note-style text.

Built-in examples

Use these to test the app before uploading your own schema.

Field inference examples

  • Admin: payer, claim ID, CPT, length of stay, total charges, readmission, discharge disposition.
  • Research: study arm, consent, enrollment, adverse event grade, protocol deviation, eligibility status.
  • Clinical: MRN, encounter, diagnosis, medication, lab test, vital sign, provider, specialty.
  • Public health: case status, pathogen, specimen, test result, vaccination status, jurisdiction, outbreak ID.
